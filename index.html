字符串

OC中新增字符串类型NSString。NSString类型的指针变量，专门用来存储OC的字符串地址。

字符串分为C字符串和OC字符串，OC的字符串常量必须使用前缀@符号,@"jjjj"是一个OC的字符串常量。

NSString *str = @"jjjj";

NSLog函数的第1个参数是OC字符串，要加@，使用NSLog输出OC字符串的值，占位符为%@

NSLog(@"aaa%@",str);

@符号：将C字符串转化为OC字符串，绝大多数关键字以@开头

数据类型

BOOL类型：可以存储YES和NO

Boolean类型：存储true和false

class类型：类

id类型：万能指针

nil：和NULL差不多

SEL：方法选择器

block：代码段

访问修饰符

默认为protected

private：只能在类的内部访问(方法是实现中访问)

protected：受保护的，只能在类和子类中访问

package：当前框架 public：公共

类的本质

(类对象)类是以Class对象的形式存储在代码段中的。 isa指针指向存储父类的类对象。

获取类对象的方法

  • SEL

    selector选择器 是个数据类型

    是一个类，SEL对象用来存储方法。

    获取SEL的方法

    SEL s1 = @selector(sayHi);

    先拿到存储方法的SEL数据(消息)，将SEL消息发送给p1对象，p1对象接收到消息后调用方法，根据对象的isa指针找到存储类的类对象，寻找类对象中是否有和传入的SEL数据相匹配的 一直寻找到父类NSObject

    手动给对象发送SEL消息

点语法

可以使用 对象名.去掉下划线的属性名，访问属性，会将点语法切换为set、get。

get和set方法中慎用点语法，可能会造成循环

@property 自动生成getter、setter方法的声明 写在interface里

自动生成一个添加下划线的属性和getter、setter的声明和实现

如果同时重写getter、setter就不会生成私有属性。

可以被子类继承，但是生成的属性是私有属性，通过getter、setter访问。

@synthesize 自动生成getter、setter的实现

@property int age; //写在声明里

@synthesize age; //写在实现中
@implementation Person
{ int age; } //自动生成一个真私有的属性
- (void)setAge: (int)age  //赋值给私有属性
{ self->age = age; } 
- (int)age   //拿到私有属性的值
{ return age; }
@end
//不会生成私有属性
@synthesize age = _age;

动态类型和静态类型

OC是弱语言。

静态类型：指针指向的对象是本类对象。动态类型：非本类对象。

编译检查：编译器在编译的时候，看指针所属的类型中是否有方法，有就会成功。

运行检查：检查对象是否有这个方法

id类型

万能指针可以指向任意OC对象，但是不能使用点语法。

id和NSObject都可以指向任意对象，但是id调用对象方法编译时直接跳过，不会做编译检查

id id1 = [Person new];

instancetype方法返回值是当前类的对象，但只能作为方法的返回值

动态语言检测

判断指针对象是否有方法

BOOL b = [p1 respondsToSelector: @selector(sayHi)]; //判断指针对象是否有方法
[Person instancesRespondToSelector: @selector(sayHi)]; //判断类方法

构造方法

new方法的内部，先调用alloc方法，在调用init方法。alloc是类方法，创建类的对象，并把对象返回。init是对象方法，初始化对象。

重写init，要先调用父类的init方法，把方法的返回值赋值给self，如果返回值是nil则父类初始化失败，初始化成功就初始化，并返回self的值。

Person *p1 = [[Person alloc] init]; //和new一样

- (instancetype)init {
    //self = [super init];
    if(self = [super init]) {  }
    return self;
}

自定义init，需要写声明

- (instancetype)initWith:(NSString *)name andAge:(int)age{
    if(self = [super init]) {
        self.name = name ;
        self.age = age;
    }
    return self;
}

Property Wrappers

属性包装器，含有@something的语句。属性包装器只是一个结构，结构中封装了应用于变量的某种模板行为。例如@State、@Published、@ObservedObject等

Published

@Published var emojiArt: EmojiArt = EmojiArt()
//具体实现
struct Published {//创建一个结构体
    var wrappedValue: EmojiArt
    //发生变化时，在信息流中发布变化
    var projectedValue: Publisher<EmojiArt, Never> //可以访问使用$emojiArt获取投影值
}
//还提供了其他变量,任何@的内容都可以制作下划线变量
var _emojiArt: Published = Published(wrappedValue: EmojiArt()) //数据类型是已发布类型
//包装之后，变成计算属性
var emojiArt: EmojiArt {
    get { _emojiArt.wrappedValue }
    set { _emojiArt.wrappedValue = newvalue }
}
